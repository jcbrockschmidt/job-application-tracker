# 1.4 New Session + Resume Generation

- [x] **`settings:getAvailableModels` IPC** — call Anthropic `models.list()`; return model IDs; cache for the process lifetime
- [x] **New Session dialog — two-phase** — modal with a large textarea for pasting the job description; Generate button triggers `sessions:create`; animated progress state (MUI LinearProgress) while company/role extraction runs; dialog closes immediately after `sessions:create` returns, before resume generation completes
- [x] **`sessions:create` IPC** — extract company name and role title from the JD via Claude (or regex fallback); create `applications` row and `sessions` row in DB; create directory `data/applications/<company>/<role>_<YYYY-MM>_<id>/`; return `Session` with `isGenerating: true` and `resume: null` — does NOT run resume generation itself
- [x] **Background resume generation** — after the dialog closes, the renderer calls `generate:resume(sessionId)`; on success, dispatches `updateSession({ resume, isGenerating: false })`; on failure, removes the session from the store and navigates away
- [x] **Loading state — sidebar** — sessions with `isGenerating: true` show a spinner and "Generating…" in the sidebar entry instead of the role title and Draft/Final badge
- [x] **Loading state — session view** — while `session.isGenerating`, the session page shows a full-screen loading indicator (spinner + company/role name) instead of the tab layout
- [x] **Abandoned session cleanup** — `sessions:getAll` (called on launch) deletes sessions where `resume.json` does not exist; these are sessions whose first generation was interrupted by an app close. Their DB rows and directories are removed automatically.
- [x] **Claude prompt: resume generation** — system prompt instructs the model to read the JD and Master CV, select and prioritize the most relevant experience, refine bullet phrasing for ATS alignment, and return a valid `ResumeJson`; all content must be grounded in Master CV bullets (no invention); log tokens to `spendLog`
- [x] **`sessions:get` IPC** — load session row + application row from DB; read `resume.json` from filesystem (if path set); read `cover-letter.json` (if path set); return `Session`
- [x] **`sessions:getAll` IPC** — return all session rows joined with application data; skip and delete sessions with no `resume.json`; used to restore sidebar on launch
