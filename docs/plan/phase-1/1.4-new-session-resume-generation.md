# 1.4 New Session + Resume Generation

- [x] **`settings:getAvailableModels` IPC** ‚Äî call Anthropic `models.list()`; return model IDs; cache for the process lifetime
- [x] **New Session dialog ‚Äî two-phase** ‚Äî modal with a large textarea for pasting the job description; Generate button triggers `sessions:create`; animated progress state (MUI LinearProgress) while company/role extraction runs; dialog closes immediately after `sessions:create` returns, before resume generation completes; the user is NOT navigated to the new session ‚Äî they remain on their current page and can continue navigating freely
- [x] **`sessions:create` IPC** ‚Äî extract company name and role title from the JD via Claude (or regex fallback); create `applications` row and `sessions` row in DB; create directory `data/applications/<company>/<role>_<YYYY-MM>_<id>/`; return `Session` with `isGenerating: true` and `resume: null` ‚Äî does NOT run resume generation itself
- [x] **Background resume generation** ‚Äî after the dialog closes, the renderer calls `generate:resume(sessionId)`; on success, dispatches `updateSession({ resume, isGenerating: false })`; on failure, removes the session from the store (no navigation, user stays on current page)
- [x] **Loading state ‚Äî sidebar** ‚Äî sessions with `isGenerating: true` show a spinner and "Generating‚Ä¶" in the sidebar entry instead of the role title and Draft/Final badge; the entry is non-clickable (disabled) until generation completes
- [x] **Loading state ‚Äî session view** ‚Äî while `session.isGenerating`, the session page shows a full-screen loading indicator (spinner + company/role name) instead of the tab layout
- [x] **Abandoned session cleanup** ‚Äî `sessions:getAll` (called on launch) deletes sessions where `resume.json` does not exist; these are sessions whose first generation was interrupted by an app close. Their DB rows and directories are removed automatically.
- [x] **Claude prompt: resume generation** ‚Äî system prompt instructs the model to read the JD and Master CV, select and prioritize the most relevant experience, refine bullet phrasing for ATS alignment, and populate a `tailor_resume` tool input matching `ResumeJson`; uses tool_use with `tool_choice: { type: "tool" }` for guaranteed structured output; all content must be grounded in Master CV bullets (no invention); log tokens to `spendLog`
- [x] **`sessions:get` IPC** ‚Äî load session row + application row from DB; read `resume.json` from filesystem (if path set); read `cover-letter.json` (if path set); return `Session`
- [x] **`sessions:getAll` IPC** ‚Äî return all session rows joined with application data; skip and delete sessions with no `resume.json`; used to restore sidebar on launch
